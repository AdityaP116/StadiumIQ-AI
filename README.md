## 2. Project Overview

**StadiumIQ** is a comprehensive, GenAI-enabled stadium operations platform originally designed for large-scale sporting events like the FIFA World Cup. 

**The Problem It Solves:** Managing a stadium with 80,000+ fans involves thousands of moving parts—from crowd bottlenecks and medical emergencies to transport logistics and accessibility routing. Traditional systems are siloed and reactive.

**What It Does:** StadiumIQ centralizes live operational data (occupancy, weather, security, medical incidents) and uses advanced Generative AI to provide real-time, actionable intelligence. It acts as an executive analyst, an emergency responder, and a dynamic fan assistant all in one.

**Who It Is Built For:** 
- Stadium Operations Directors & Security Personnel
- Medical Response Units & Crowd Management Teams
- Event Attendees & Fans requiring smart navigation

## 3. Features

* 🤖 **AI Fan Assistant**: Natural language Q&A for attendees.
* 🗺 **Smart Navigation**: Dynamic, step-by-step indoor/outdoor routing using natural landmarks.
* 👥 **Crowd Management**: Real-time risk assessment and AI-generated diversion tactics based on live gate occupancy.
* 🚨 **Emergency Response**: Automated, AI-generated emergency response plans logged directly for audit trails.
* 🌍 **Multilingual Support**: AI-generated public address (PA) announcements in multiple languages.
* ♿ **Accessibility Assistance**: Specialized routing and assistance planning for wheelchair users and visually impaired fans.
* 🚉 **Transport Recommendations**: Live transport advisories based on current traffic and metro delays.
* 🌱 **Sustainability Insights**: Eco-friendly operational suggestions dynamically generated by AI.
* 📊 **Live Operations Dashboard**: Executive AI-summarized insights running alongside real-time Socket.io metrics.

## 6. Architecture

StadiumIQ relies on a clean, scalable architecture separating presentation from business logic and AI processing. The backend strictly ensures that live operational data is collected *before* being injected into AI prompts to prevent hallucinations.

```text
Frontend (React/Vite)
       ↓
REST API (Axios)
       ↓
Express Server (Controllers & Rate Limiting)
       ↓
Feature Services (Live Data Fetching)
       ↓
Prompt Builders (Context Injection)
       ↓
OpenAI Client SDK (Configured for NVIDIA NIM)
       ↓
Firebase (Firestore Logging & Auth)
       ↓
Real-time Socket.io Updates
```

## 8. Installation

Follow these steps to run the project locally.

### 1. Clone the repository
```bash
git clone https://github.com/yourusername/stadiumiq-ai.git
cd stadiumiq-ai
```

### 2. Install Dependencies
Open two terminal windows (one for frontend, one for backend).
```bash
# In terminal 1
cd backend
npm install

# In terminal 2
cd frontend
npm install
```

### 3. Configure Environment Variables
Copy the example files and populate them with your keys.
```bash
# Backend
cd backend
cp .env.example .env

# Frontend
cd frontend
cp .env.example .env
```

### 4. Run Backend
```bash
cd backend
npm run dev
```

### 5. Run Frontend
```bash
cd frontend
npm run dev
```

## 11. Usage

1. **Dashboard Overview:** Upon logging in, administrators see a live dashboard driven by real-time Socket.io updates showing medical incidents, crowd capacity, and weather.
2. **AI Insights:** Clicking "Generate Insight" triggers the backend to bundle current live data, inject it into a dynamic prompt, and fetch a JSON-formatted executive summary from the NVIDIA AI model.
3. **Fan App:** Attendees can ask the AI assistant questions. The system processes their location to give contextual answers regarding the nearest food, exits, or transport options.
4. **Emergencies:** If an incident is reported, the system generates a structured, step-by-step response plan which is permanently logged in Firestore for accountability.

## 12. Project Workflow

1. **Data Collection:** `fakeDataGenerator.js` (or live sensors) creates a snapshot of the stadium's current state.
2. **Action Trigger:** A user (Fan or Admin) makes a request via the React frontend.
3. **Routing:** The Express controller validates the request and routes it to the specific Feature Service (e.g., `crowd.service.js`).
4. **Context Building:** The service fetches the live data and passes it to a dedicated Prompt Builder.
5. **AI Inference:** The OpenAI SDK makes an external call to NVIDIA NIM APIs, enforcing JSON outputs.
6. **Delivery:** The structured response is parsed, logged in Firebase if necessary, and sent back to the frontend UI.

## 13. AI Integration

- **Provider:** The project uses the **OpenAI Node.js SDK**, configured to point to the **NVIDIA NIM API** (`baseURL: https://integrate.api.nvidia.com/v1`).
- **Model:** Currently utilizing the `openai/gpt-oss-20b` model.
- **Prompt Engineering:** Prompts are stored cleanly in the `src/prompts/` directory. Static text is combined with dynamic JSON representations of live stadium state to prevent hallucinations.
- **Response Handling:** The AI service (`openai.service.js`) handles rate-limiting, exponential backoff retries, extracts `reasoning_content` (for Chain-of-Thought models), and strictly parses JSON responses before returning them to the application logic.
